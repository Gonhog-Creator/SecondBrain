# ---
# RAILWAY SETUP INSTRUCTIONS
#
# Step 1: Create Project
# - Go to Railway dashboard
# - Click "New Project" -> "Deploy from GitHub repo"
# - Select your DeltaDash repository
#
# Step 2: Add Postgres Database
# - Click "New Service" -> Select "Postgres"
# - Railway will automatically inject DATABASE_URL environment variable
#
# Step 3: Configure Backend Service (already created from repo)
# - Go to your backend service settings
# - Add these environment variables:
#   * SECRET_KEY: Generate with: python -c "import secrets; print(secrets.token_hex(32))"
#   * CORS_ORIGINS: Set to your frontend URL (e.g., https://your-frontend.railway.app)
#   * APP_ENV: production
#   * USE_RAILWAY_STORAGE: true
#
# Step 4: Add Frontend Service
# - Click "New Service" -> "Deploy from GitHub repo"
# - Select the same repository
# - In the service settings:
#   * Root Directory: frontend
#   * Dockerfile Path: frontend/Dockerfile
#   * Build Args: VITE_API_URL=https://your-backend-service.railway.app
# - Or add VITE_API_URL as environment variable (frontend Dockerfile will pick it up)
#
# Step 5: Connect Services
# - Once both services are deployed, copy their URLs
# - Update CORS_ORIGINS on backend with frontend URL
# - Update VITE_API_URL on frontend with backend URL
#
# Step 6: Database Users
# - Railway Postgres creates a default user automatically
# - The DATABASE_URL includes all credentials (user, password, host, port, db)
# - No manual user creation needed - Railway handles this
# - Alembic migrations will run automatically via the startCommand
#
# Troubleshooting:
# - If build fails: Check Railway logs for specific error
# - If migrations fail: Ensure DATABASE_URL is set correctly by Railway
# - If CORS errors: Verify CORS_ORIGINS matches your frontend URL exactly
# ---
